# Subject Slide Maker

중·고등학교 교과서 내용을 **교과별 전용 스타일**의 웹 슬라이드로 변환하는 스킬.
교과서 PDF나 텍스트를 받아, 단원 구조를 따라가며 빠르게 넘기는 수업용 슬라이드를 만든다.

지원 교과: **기술 · 윤리 · 영어 · 국어 · 일본어 · 과학 · 진로 · 사서 · 역사 · 사회** (10교과).
역사는 한국사·동양사·서양사를 **하나의 공용 세피아 테마**로 지원한다(`themes.md` 9번 + `subject-types.md` 역사 전용 틀). 역사 전용 단일 파일 스킬 `history-slide-maker`도 그대로 남아 있으나, 이제 본 스킬 하나로 역사까지 만들 수 있다.
사회는 통합사회·정치와 법·경제·사회·문화·지리를 **하나의 공용 테마**로 폭넓게 지원한다(`themes.md` 10번 + `subject-types.md` 사회 전용 틀).

## 이 스킬의 구성 (필요할 때 해당 파일을 읽어라)

- **`references/themes.md`** — 8교과 각각의 색 팔레트·폰트 짝·모티프(CSS 변수). 덱 시작 시 **교과 하나의 변수 블록을 복사**한다.
- **`references/smartart.md`** — PPT SmartArt식 순수 CSS 컴포넌트 11종(프로세스·사이클·계층·피라미드·벤·매트릭스·퍼널·허브-스포크·타임라인·비교·막대). 테마 변수만 참조하므로 색이 자동 적용된다.
- **`references/subject-types.md`** — 교과마다 그 교과서에만 나오는 전용 자료 틀(예: 과학 실험 과정도, 국어 작품 인용, 일본어 가나표, 윤리 딜레마 저울).
- **`scripts/extract_pdf.py`** — 교과서 PDF에서 텍스트·이미지 자동 추출.

## 작업 순서

1. **교과 확인** → `themes.md`에서 그 교과 CSS 변수 + 폰트 `@import`를 가져온다.
2. 입력이 PDF면 **추출 스크립트 실행**(아래), 텍스트면 바로 분석.
3. **단원 구조 파악** → 슬라이드 시퀀스로 매핑.
4. 교과서 서술을 **쪼개고 분석**해 `smartart.md`·`subject-types.md`의 틀로 재구성.
5. 단일 HTML로 출력, **브라우저로 렌더 검증**.

## 핵심 전제

- **중·고등학교** 수업용
- **단원별** 슬라이드 (한 차시가 아님)
- **빠르게 넘기는** 스타일 — 학생 필기 시간 없음
- **교과서 단원 구조** 그대로 따라감
- 교과서 텍스트를 **최대한 쪼개고 분석**해서 시각적으로 재구성
- 교과서 이미지/그래프/지도는 **캡쳐해서 그대로** 사용 (저작권 OK)

### 교과서 자료 빠짐없이 활용 (중요)

본문만이 아니라 교과서의 **모든 요소**를 슬라이드에 흡수한다: 양 날개의 작은 글씨, 사료/자료 박스, 탐구·활동, 그래프·표 캡션, 지도·사진 설명, 각주, 용어 풀이, 단원 도입 질문, 정리 코너까지. 교과서 페이지에 있는 텍스트와 자료를 **하나도 버리지 말고** 적절한 컴포넌트(스마트아트·전용 틀)에 배치한다. "본문만 옮기고 날개·캡션을 버리는 것"은 이 스킬의 실패다.

### SmartArt 적극 활용 (중요)

나열·순서·분류·비교·위계·순환 같은 구조가 보이면 **글머리표로 늘어놓지 말고** `smartart.md`의 컴포넌트로 시각화한다. "컴포넌트 고르기 빠른 가이드"(smartart.md 하단)를 기준으로:
순서→프로세스, 순환→사이클, 분류/조직→계층, 기초→정점→피라미드, 공통점→벤, 두 기준→매트릭스, 걸러짐→퍼널, 중심+요소→허브, 시간순→타임라인, 대조→비교, 수치→막대.

## 기본 제약 (web-slide-presenter 공통)

- 모든 슬라이드: `100vw × 100vh`, `overflow: hidden`, 스크롤 없음
- **슬라이드 콘텐츠는 화면 중앙 정렬**이 기본:
```css
.slide {
  width: 100vw; height: 100vh; overflow: hidden;
  display: flex; flex-direction: column;
  align-items: center; justify-content: center;
  padding: 5vh 8vw;
  background: var(--bg); color: var(--text);
}
.slide.dark { background: var(--bg-dark); color: var(--text-dark); }
```
- 콘텐츠 넘치면 **슬라이드를 쪼개라** (폰트 줄이지 마라)
- 순수 바닐라 JS, 외부 라이브러리 금지 (웹폰트 CDN 허용)
- 키보드 네비게이션: →/↓/Space = 다음, ←/↑ = 이전
- 단일 HTML 파일 (이미지는 별도 폴더)
- vw/vh 혼용 금지 — 원·정사각형 등 비율 중요한 요소는 한 단위(vh)만
- **한국어 단어 줄바꿈 방지** (모든 슬라이드 필수):
```css
* { word-break: keep-all; overflow-wrap: break-word; }
```

## 글자 크기 (교실 뒷자리 가독성 — 교과 공통)

| 요소 | 최소 | 권장 |
|------|------|------|
| 단원/슬라이드 제목 | 4vh | 5~7vh |
| 소단원 제목 | 3vh | 3.5~4.5vh |
| 핵심 키워드/숫자 | 3vh | 4~6vh |
| 본문/설명 | 2vh | 2~2.6vh |
| 캡션/출처 | 1.4vh | 1.5~1.8vh |

색·폰트는 교과 테마(`themes.md`)를 따르되, 위 크기 기준은 교과 불문 공통이다.

## 입력 처리: 교과서 PDF

```bash
python <skill-dir>/scripts/extract_pdf.py <pdf경로> <출력폴더> --pages <범위>
```
자동 생성: `text.txt`(페이지별 텍스트), `images/`(`p3_img0.jpeg` 형태, 50px 미만 아이콘 스킵), `image_manifest.json`(이미지 목록).

이후: 텍스트 분석 → `image_manifest.json`으로 이미지 매칭(`<img src="images/p3_img0.jpeg">`) → 단원 구조 파악 → 슬라이드 생성.
이미지 슬라이드는 `object-fit: contain`으로 비율 유지, 100vh 안에 맞춤. base64 인라인은 5장 이하일 때만.

## 슬라이드 시퀀스: 단원 구조 → 슬라이드

1. **단원 표지(1장)** — 대단원 번호+제목 크게. 배경 다크(`--bg-dark`). 대표 이미지 있으면 풀블리드 + 텍스트 오버레이.
2. **학습목표(1장)** — 교과서 학습목표를 프래그먼트로 순차 표시.
3. **소단원별 본문** — 소단원 시작은 다크 구분 슬라이드. 이후 교과서 내용을 쪼개 슬라이드화(아래 재구성 규칙). **교과서 1페이지당 2~4장** 예상.
4. **단원 정리(1~2장)** — 핵심 키워드 정리 또는 교과서 "단원 정리" 시각화.

## 텍스트 재구성 규칙

교과서 서술을 읽고, 가장 적합한 형태로 변환한다. **텍스트를 그대로 옮기지 마라 — 쪼개고, 분석하고, 시각적으로 재구성하라.**

### 6×6 원칙
- 한 슬라이드에 **키워드 6개 이하**, 한 줄에 **6단어 이하**
- 문장이 아닌 **키워드** 중심. 설명이 필요하면 본문 폰트로 짧은 서술 1~2줄.

### 패턴 매칭 (→ 컴포넌트)
| 교과서 서술 유형 | 컴포넌트 |
|---|---|
| 절차·공정·순서 | 프로세스 |
| 순환·반복 | 사이클 |
| 분류·조직·상하 | 계층/조직도 |
| 기초→정점 위계 | 피라미드 |
| 공통점·교집합 | 벤다이어그램 |
| 두 기준 분류 | 매트릭스 |
| 걸러짐·선별 | 퍼널 |
| 중심 개념 + 요소 | 허브-스포크 |
| 연·월·단계 시간순 | 타임라인 |
| 둘 비교 | 비교 |
| 수치 데이터 | 막대그래프 |
| 교과 특수 자료 | `subject-types.md`의 전용 틀 |

### 복잡한 서술
- 3문장 이상 단락 → 무조건 쪼개 2장 이상으로
- 하나의 개념 = 하나의 슬라이드. 한 슬라이드에 두 개념·두 컴포넌트 금지
- 쪼갤 때: 서술의 뼈대(시간순/인과/위계)를 먼저 파악하고 키워드 2~3개씩 시각 요소로 분배

## 프래그먼트 (빌드 효과)

- 같은 레벨 항목 나열(카드·리스트·노드·층) → 순차 등장
- 프로세스/사이클 단계, 비교 양쪽, 매트릭스 사분면 → 하나씩 공개
- **쓰지 마라**: 단원 표지, 소단원 구분, 이미지 풀블리드

```js
// 표준 네비게이션 + 프래그먼트
const slides=[...document.querySelectorAll('.slide')]; let cur=0;
function frags(s){return [...s.querySelectorAll('.fragment:not(.visible)')];}
function next(){const s=slides[cur]; const f=frags(s);
  if(f.length){f[0].classList.add('visible');return;}
  if(cur<slides.length-1){slides[cur].classList.remove('active');slides[++cur].classList.add('active');}}
function prev(){const s=slides[cur]; const v=[...s.querySelectorAll('.fragment.visible')];
  if(v.length){v[v.length-1].classList.remove('visible');return;}
  if(cur>0){slides[cur].classList.remove('active');slides[--cur].classList.add('active');}}
addEventListener('keydown',e=>{
  if(['ArrowRight','ArrowDown',' '].includes(e.key)){e.preventDefault();next();}
  else if(['ArrowLeft','ArrowUp'].includes(e.key)){e.preventDefault();prev();}});
slides[0].classList.add('active');
```
```css
.slide{display:none;} .slide.active{display:flex;}
```

## 금지 / 주의

- **동심원(겹치는 여러 원) 금지.** 사이클은 단일 링 위 노드 배치, 벤은 `mix-blend-mode`(smartart.md 참고).
- 이모지 금지. AI 클리셰(순수 블랙 + 네온 퍼플/그린) 금지 — 교과 테마 색을 쓴다.
- hex 직박기 금지 — 색은 전부 `themes.md`의 CSS 변수로.
